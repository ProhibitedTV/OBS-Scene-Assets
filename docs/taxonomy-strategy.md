# Scalable Asset Taxonomy

This repo is aiming beyond a small template pack. The long-term goal is a massive white-label OBS library with thousands of animated assets that cover thousands of discoverable creator niches without turning the gallery into chaos.

The key is to separate **stable top-level structure** from **high-volume long-tail coverage**.

## 1. Keep Asset Types Stable

`Category` should stay a compact set of asset jobs that are easy to browse in the gallery:

- `alerts`
- `backgrounds`
- `lower-thirds`
- `news`
- `overlays`
- `scenes`
- `transitions`
- `widgets`

Do not create a new top-level category just because a niche is new. "Podcast", "charity", "esports", or "holiday" are better expressed through collections, folders, packs, and tags.

## 2. Use Collections For Repeatable Themes

`Collection` is where the repo should expand aggressively.

Use collections for:

- audience-specific suites
- genre or franchise packs
- visual systems and brand languages
- platform-first workflows
- event-ready bundles

Examples:

- `live-show`
- `vertical-creator`
- `games/halo`
- `games/valorant`
- `newsroom`

If a new idea represents a reusable visual family that could support many assets, it is a good candidate for a collection.

## 3. Ship Packs, Not Random Singles

A scalable repo should release coherent packs that creators can actually use end-to-end.

A healthy pack usually includes:

- alerts
- lower thirds
- overlays
- scenes
- transitions
- optional widgets or backgrounds

That approach turns one niche idea into 12-20 usable assets instead of one isolated file.

## 4. Use Tags For Long-Tail Discovery

Tags are where "thousands of categories" become practical.

Tags should capture discoverability cues such as:

- broadcast format: `podcast`, `interview`, `watchalong`, `just-chatting`
- platform or layout: `vertical`, `mobile`, `tiktok`, `shorts`
- visual mood: `clean`, `cozy`, `cinematic`, `retro`, `esports`
- event type: `launch`, `charity`, `holiday`, `tournament`, `finals`
- subject matter: `music`, `news`, `sports`, `fantasy`, `sci-fi`
- workflow/job: `countdown`, `schedule`, `sponsor`, `goal`, `scoreboard`

Thousands of discoverable niches should come from combinations of collection + category + tags, not from exploding the top-level category list.

## 5. Coverage Axes To Expand Against

To grow toward thousands of assets, keep adding packs across multiple axes:

- creator format
- game or content genre
- event type
- monetization or campaign use case
- visual style and mood
- platform or aspect ratio

Example combinations:

- vertical + podcast + sponsor
- halo + tournament + analyst desk
- cozy + just-chatting + schedule board
- charity + donation + goal tracker
- newsroom + election + ticker

Each combination can become a coordinated mini-catalog rather than a one-off asset.

## 6. Decision Rules

When adding something new, use this quick test:

- New broadcast job? Keep the existing asset type if possible.
- New reusable visual family? Add or extend a collection.
- New coordinated release theme? Build a pack.
- New long-tail search cue? Add tags.
- New top-level category? Only if the current eight asset types genuinely cannot describe the job.

## 7. Suggested Growth Lanes

These parent lanes are broad enough to generate hundreds of packs over time:

- charity and fundraising
- cozy and lifestyle
- education and coaching
- esports and tournament desk
- general creator essentials
- IRL and travel
- music and DJ
- news and politics
- podcast and live show
- sports and watchalong
- vertical and mobile-first
- VTuber and avatar-first

## 8. Operational Implication

If the repo is going to scale well, every contributor should optimize for two things at once:

1. Increase coverage breadth with new packs, themes, and niches.
2. Preserve a small, legible information architecture so creators can still find the right asset quickly.

That balance is what makes "thousands of assets across thousands of niches" realistic instead of messy.
